General

top

Content-Types

The API provides the following content types

  • application/json: JSON data to be used as raw information input.
  • application/pdf: A PDF report for timeseries
  • image/png: A rendered representation of one or multiple timeseries.

Paging

Resource collections are paged by default to restrict large data access (configurable by the API provider .. use size=MAX to get the maximum of a page). The query parameters offset and size can be used to define exactly that part of the collection to be returned by the server.

Considering the Web Linking (RFC5988) approach the exact location within a collection can be retrieved from response header information.

Pageing Example

GET /api/v1/stations?offset=4&size=2 HTTP/1.1
	Host: sensorweb.demo.52north.org
	Accept: application/json
HTTP/1.1 200 OK
	Content-Type: application/json;charset=UTF-8
	Link: </api/v1/stations?offset=4&size=2>; rel="self"
	Link: </api/v1/stations?offset=0&size=2>; rel="first"; title="first page"
	Link: </api/v1/stations?offset=2349&size=2>; rel="last"; title="last page"
	Link: </api/v1/stations?offset=2&size=2>; rel="prev"; title="previous page"
	Link: </api/v1/stations?offset=6&size=2>; rel="next"; title="next page"

	Content omitted for brevity.

URLs

API calls are triggered by using the following URLs.

Method Relative Path Description
GET /api/v1/stations Lists all measured stations available.
GET /api/v1/stations/{id} Lists the measured stations with id {id}.

Status Codes

HTTP Status codes indicate the following states.

Code When
200 Status OK. All went fine.
404 Status Not Found. The resource is not available.
500 Status Internal Server Error. Please contact the webadmin. In some cases an error message will be shown. But in general the server logs will help to trace the problem.

Query Parameters

Stations can be queried with the following filter parameters.

Parameter Example Description
service
 
service=srv_sdlfj2
 All stations where the given service id do match.
category
 
category=cat_sadf2d
 All stations where the given category id do match.
offering
 
offering=off_fdoi23
 All stations where the given offering id do match.
feature
 
feature=foi_adf092e
 All stations where the given feature id do match.
procedure
 
procedure=pro_asd23
 All stations where the given procedure id do match.
observedProperty 
observedProperty=phe_sdjf210
 All stations where the given observed property id do match.
near 
near={
  "center": { 
    "type": "Point", 
    "coordinates": [<lgt>,<lat>]
  }, 
  "radius":"500" 
}

Stations around a given point. The radius unit is in kilometers (km).

If you want a different CRS than the default (WGS84) you can add your own to the GeoJSON geometry, e.g. 'crs':{'type':'name','properties'{'name':'EPSG:25832'}}.
within 
within={
  'll': { 
    "type": "Point", 
    "coordinates": [<lgt>,<lat>]
  },
  'ur': { 
    "type": "Point",
    "coordinates": [<lgt>,<lat>]
  } 
}

Stations within a given bounding box.

If you want a different CRS than the default (WGS84) you can add your own to the GeoJSON geometry, e.g. 'crs':{'type':'name','properties'{'name':'EPSG:25832'}}.

Timeseries

URLs

API calls are triggered by using the following URLs.

Method Relative Path Description
GET /api/v1/timeseries Lists all timeseries available.
POST /api/v1/timeseries Request to obtain multiple timeseries. Set the Accept: application/json header to obtain JSON data, or set Accept: image/png to obtain a rendered graph displaying the timeseries with requested style options
document request POST structure with style options.
GET /api/v1/timeseries/{id}

Lists the timeseries with id {id}. Set the Accept: application/json header to obtain JSON data, or set Accept: image/png to obtain a rendered graph displaying the timeseries with requested style options.

Document query parameter style. Also available refValues must be stylable.

Status Codes

HTTP Status codes indicate the following states.

Code When
200 Status OK. All went fine.
400 Status Bad Request. The request data cannot be read.
404 Status Not Found. The resource is not available.
500 Status Internal Server Error. Please contact the webadmin. In some cases an error message will be shown. But in general the server logs will help to trace the problem.
