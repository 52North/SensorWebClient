1. Introduction

This client allows subscribing to notifications in case of certain sensor measurements or combinations of measured values. The system ist based on the OGC Sensor Event Service.
The development of this client has been performed using the Google Web Toolkit. Thus, this clients consists mainly of JavaScript code.
The user behaviour is not analysed by this client. Furthermore, no data are sent to Google.

Short description of available buttons in the buttons bar.
The visibility depends on user role: not registered user/registered user/administrator.

Not registered user:

Register Opens the register view where user can register to SES client
Help Open the Help Window

3. Views

Login


This is the default view when the user starts the SES Client. The user can login, register or let the system send him a new password if he forgot it.

Registration


By filling all required fields the user can register to the client. After the registration is completed the user receives an enail to verify his registration.

Forgot your password


In case user forgot his password, he can use this option to request a new password from the client.

Subscribe to rules


This is the default view when the user logs in. This view has two tables where the user can see all his rules and rules published by other users. Here the user can subscribe to rules.

Edit rules


Here the user can edit, publish or delete his (user is the owner/creator) rules.

Create basic rules


Here user can create simple rules. The user has the choice between five different types of rules:
  1. Overshoot/Undershoot
  2. Trend over tme
  3. Trend over count
  4. Sum over time
  5. Sensor failure
Inputs are case sensitive!

Create complex rules


Here the user can create complex rules. By adding a block the user can add two rules to the combination.

Subscriptions


A list of all subscriptions of the current user is displayed. The user can unsubscribe from each rule.

Edit profile


Here the user can change his personal data like email address or password.

User management


A list of all registered users in the SES Client is displayed. The administrator can edit, delete or create new user.

Sensor management


A list of all registered sensors in the SES Client is displayed. The administrator can see whether sensors are in use and can activate or deactivate them. Deactivated sensors are not listed in the list of possible sensors for creating basic rules.

Show all rules


A list of all created basic and complex rules is displayed. The administrator can see published and not published rules and delete or publish them.
